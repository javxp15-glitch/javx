Sync videos from the JAVX API into WordPress and RetroTube with token auth, full-import batching, progress tracking, and WP-Cron incremental updates.

== Description ==

JAVX RetroTube Sync is a WordPress plugin that imports remote videos from the JAVX API into WordPress posts.

Features:

* API token authentication
* Dedicated plugin endpoints (`/api/plugin/videos`, `/api/plugin/videos/:id`, `/api/plugin/videos/sync`)
* Incremental sync via WP-Cron
* Manual full import with progress bar and recent logs
* Idempotent upsert using source video id and source updated timestamp
* Configurable RetroTube meta mapping
* Optional featured image sideloading
* Optional `pornstar` taxonomy registration

== Installation ==

1. Upload `javx-retrotube-sync-1.0.0.zip` in `Plugins > Add New > Upload Plugin`.
2. Activate the plugin.
3. Open `JAVX Sync` in the WordPress admin menu.
4. Enter:
   * API Base URL, for example `https://source.example.com`
   * API Token generated by the JAVX admin
   * RetroTube mapping keys for playback or embed mode
5. Save settings.
6. Click `Test Connection`.
7. Run `Start Full Import` for the first sync.

== Configuration ==

Recommended settings for first production run:

* Batch size: `20`
* Sync interval: `Every 10 minutes`
* Import mode: `Playback URL` if RetroTube accepts remote MP4 URLs
* Target post type: `post` unless your RetroTube install uses a different video post type

== API Contract ==

The plugin expects the source site to expose:

* `GET /api/plugin/videos`
* `GET /api/plugin/videos/:id`
* `GET /api/plugin/videos/sync`

All requests must accept `Authorization: Bearer <token>`.
